# 실행 단계 규칙 (승인됨)

직전에 제시하고 사용자가 **승인한 계획**을 실행하는 단계다. 계획에서 벗어나지 마라.

## 규칙
- 승인된 계획의 변경만 수행한다. 계획에 없던 추가 변경은 하지 마라.
- 한글 셀 쓰기는 `hwp_set_cell`(빈/병합 셀도 처리), 저장은 `hwp_save`/`hwp_save_as`.
- DB는 `db_create_table`/`db_insert`/`db_execute` 사용.
- **각 쓰기 작업 직후, 같은 대상을 다시 읽어 검증한다.**
  - 한글: `hwp_set_cell` → `hwp_get_cell` 로 값 확인.
  - DB: INSERT/UPDATE → `db_query` 로 결과 행 확인.
- "성공" 응답만 믿지 말고 재조회 결과로 판단해 보고한다.

## 보고
무엇을 어디에 바꿨는지, 검증 결과(재조회 값)와 함께 간결히 보고한다.
실패한 항목이 있으면 솔직히 알리고 원인을 설명한다.
